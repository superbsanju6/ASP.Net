Roster File
What is a Roster File?

A roster file is a file that contains all the teacher, student and class information that needs to be loaded to your system.  A roster file is the most important of the three major data files that need loaded to your system.  The roster file sets up all the classes and adds all the teachers and students to your system.  It is very important that roster files contain as much information as possible and follow a consistent format in order to prevent the creation of duplicates or incorrect data being loaded.
What information is required in a Roster File?

A roster file contains all the information related to teachers, students and classes.  The file includes student demographics as well as any additional user defined fields that the user wishes to track.  Listed below are the required and optional column headers as well as a description of a few of the columns.  Red text denotes required and blue denotes optional.

***Teacher ID*** - This field is not required, however, we highly recommend sending unique teacher id's for your teachers.  Teacher ID's prevent the creation of duplicate users if a teacher were to switch schools or change names.

Teacher Email

Teacher User ID - This is what the teacher will use to log into the system.  If the column is left blank then a default firstinitial.lastname (i.e. John Smith, Jsmith) user ID will be created.

What format does a Roster File need to be submitted in?

Roster files needs to be submitted as comma delimited (.csv) files.
How do I send a Roster File to Thinkgate?

In order to send a Roster File to Thinkgate Support you must have tmanager rights in your system.  If you are a tmanager, listed below are directions:

 

1.Click Support
2.Click Upload Files to Support
3.Click Browse in the resulting pop-up
4.Located the file on your computer
5.Double click the file once located
6.Click Continue
How long does it take to get a Roster File uploaded?

Once a roster file has been received by Thinkgate Support, minus any import issues or if the file was submitted on a Friday, the file will be uploaded within 48 hours.  If there are any import issues with the file the import process could take longer.  If the issues cannot be resolved by Thinkgate Support you will be contacted any may be required to re-send the file after changes are made.  If the file is submitted on a Friday it will be uploaded at the beginning of the following week.  If an issue is found in the import and Thinkgate Support makes the request for a new file or correction to the existing file the 48 hour time period will start again as soon as the corrections are made or new file received.
How often do I need to send a Roster File?

Roster files can be sent whenever the customer desires.  If your system experiences constant roster changes you may wish to send the file more frequently.  However, if you do not have many changes it is not required that you send the roster file.  The roster file is designed to keep your system up to date and mirror your Student Information System.  Therefore, if you experience a lot of changes in your SIS you may send a new roster file.

 

Admin File
What is an Admin File?

The Admin File contains all users other then teachers that need loaded to your system.  This includes administrators, cmanagers, tmanagers and any other custom user types.

What information is required in an Admin File?
