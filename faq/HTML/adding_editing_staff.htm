What are the different user types and what is the difference?

The 4 default user types are Tmanager, Cmanager, Administrator and Teacher.  Listed below are the default restrictions and permissions for the different user types.  If your district uses customer user types then many of these restrictions could be different.  You will need to contact your district administrator or Thinkgate Support (if permitted) for more information.

 

1.Tmanager - Tmanagers are the least restricted, highest user type in a system.  The tmanager user type is usually reserved for the Primary Point and Data Contact in a district.  Tmanagers have access to all modules purchased by a district as well as the ability to upload files to Thinkgate Support.  Tmanager do not have any school restrictions.
 
2.Cmanagers - Cmanagers are the second highest user type.  They have all the same access as the tmanager minus the ability to upload files to Thinkgate Support.  By default, cmanagers have no school restrictions.  This user type is usually reserved for a district's assessment building team or permitted users needing to run district wide reports.
 
3.Administrators - Adminstrators are the "read only" user type in Elements.  They are restricted to their respective school and can report on any data at the school level.  This user type is usually restricted for principals at individual schools who only need to run reports on their school.
 
4.Teachers - The teacher user type is unique in Elements due to the fact that all criteria is controlled by a class and curriculum drop down in the left nav.  A teacher can only see data based on the classes they are tied to.  They control which class they want to look at by manipulating the two drop downs in their left nav.  Teachers do not have access to any school or district information.
Can I create custom user types?

Yes, a district has the option to create custom user types or can request specific permissions for a user type that is already available in Elements.  If custom users need to be created please contact your district administrator or Thinkgate Support.
Can I change a user's user type after they are added to the system?

Yes, once a user has been added to the system their user type can be changed at any time.  This can only be done by tmanager or cmanagers.  Please follow the steps below to change a user's user type:

 

1.Click Support
2.Select the Find Staff Icon
3.Enter the criteria to search for the desired user
4.Click the Edit icon for the desired user
5.Use the User Type drop down to select a new user type for the user
6.Click Save
Can a user be deleted from the system?

Yes, a user can be deleted from the system. However, in the case of teachers, if they have any current or prior year classes or data tied to them they cannot be deleted.  If a teacher leaves the school but has historical data they must remain in the system.  This is due to prior year reporting purposes. In order to delete a user from the system please follow the directions listed below:

 

1.Click Support
2.Click the Find Staff icon
3.Enter the criteria to search for the desired user
4.Click the Edit icon for the desired user
5.Select Delete in the left nav
Can I lock a user out of the system without deleting them?

Yes, a user can be locked out of the system by revoking their access.  In order to revoke a user's access follow these steps (only tmanagers and cmanagers can revoke a user):

 

1.Click Support
2.Click the Find Staff Icon
3.Enter the criteria to search for the desired user
4.Click the Edit icon for the desired user
5.Change the Restrictions drop down to Access Revoked
6.Click Save

 

Once a user is set to Access Revoked they will be unable to log into the system.  The revoked status can be removed at anytime.
I just added a new user, however, they are not showing when I search for them.  Why?

When a teacher has been added but cannot be located it is normally due to the status drop down in the search criteria being set to current year.  Teachers who do not have current year classes can be found by changing this drop down to All Years.  In order to make this change follow these steps:

 

1.Click Support
2.Select  the Find Staff icon
3.Use the criteria to search for the desired user. Make sure you change the User Type drop down to Teachers, this will give you access to the Status drop down.
4.Change the Status drop down to All Years
5.Select Find Staff

 

The teacher should now be included in the list.  New teachers will also show up in the list if the user type is set to all because under this option there is no Status drop down.
When a teacher signs in they do not see any classes, why?

There are two possible issues if a teacher logs in and does not see any classes.

•The teacher is not assigned to any classes.  If a teacher is manually added then not associated with any classes they will not see any drop downs when they log in.
 
•A duplicate user was created, either manually or through a roster import.  If a teacher has a duplicate user then one user will usually have the classes and the other will not.  In order to check if a teacher has a duplicate user, follow these steps:
1.Click Support
2.Choose the Find Staff icon
3.Use the criteria to search for the desired user. Try be use the least amount of criteria possible, for example, only the last name or first three letters of last name.
4.Click Find Staff

 

If a user has duplicate logins you should see their name twice in the resulting list (or some form of their name).  If you see duplicate users you will need to contact your district administrator or Thinkgate Support (if permitted).
How are duplicate users created?

There are two ways a duplicate user can be created.  Either through manually adding a user or a roster import.

 

1.If a user is manually added with a different name, loginID, school, etc. then their record in the district's Information System then when a file is imported a duplicate user will be created.
 
2.If a user's record in a roster file changes between two different files, for example, new school, last name, first name, user id, etc. (and teacher ID's are not included) then a duplicate will be created.

 

The best way to prevent duplicate users from being created is to include a Teacher ID column in the roster file.  This teacher ID can be used as a unique identifier for every teacher and can be used as a cross reference to update existing teacher records.  In the case of manually added staff, the manually added user needs to be deleted since, more than likely, classes will be associated with the other user.  If you run into any issues with duplicate users, or see a lot of them in your system, contact your district administrator or Thinkgate Support (if permitted).
What do I do if I see duplicate users?

If you find duplicate users in your system please contact your district administrator or Thinkgate Support (if permitted).
